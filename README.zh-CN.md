**Augur** 召集了十八位投资史上的传奇人物——巴菲特、芒格、格雷厄姆、索罗斯、达利欧、西蒙斯、凯茜·伍德、迈克尔·伯里等齐聚一堂，完全按照各自的投资哲学和表达风格，为你输入的标的给出独立判断。各位大师互不迁就，也不会为了达成共识刻意折衷观点。最终由专属「编审」汇总所有投票结果、梳理核心分歧，输出最终研判结论。

你拿到的不是一个干巴巴的结论，而是支撑这个结论的全部思考过程。

**本项目不构成任何投资建议，仅供研究与娱乐使用。**

---

## 为什么是议事会，不是单一神谕？

随便找个通用大模型问某只股票怎么样，你得到的只会是互联网所有观点的平均产物——模棱两可、四平八稳、转头就忘。投资里真正有价值的信号从来不在平均值里，而在那些思考方式截然不同、却都在市场里反复证明过自己判断力的人之间的分歧里。

Augur 把这些分歧原汁原味地摆到了你面前。巴菲特和凯茜·伍德看同一份财报，可能会得出完全相反的结论——不是谁能力不够，而是他们的底层投资信仰根本不一样。十八位大师，十八种观察视角，同一个标的。最终的研判结论，是你只有把这些人放在同一个房间里辩论才能看到的东西。

## 三幕流程

### 第一幕 · 问卜（The Auspices）

专属研究模型会生成4-6条精准搜索查询——都是专业分析师拿到标的后第一时间会去核实的问题，交由Exa或Tavily并行执行。所有搜索结果会被压缩成一份中立客观的**市场快照**：公司主营业务是什么、近期股价走势如何、在行业里处于什么位置、当前宏观环境怎样。

快照里的每一个论断都基于实时网页搜索结果，永远不会只用模型陈旧的训练数据做判断。

### 第二幕 · 议事（The Council）

市场快照会同步发送给所有十八位大师，每个人拿到的都是*完全相同*的简报，然后以自己的标志性风格返回一份 `PersonaVote`：**买入/持有/卖出**建议、时间周期、推荐仓位、核心理由、主要担忧，以及2-3段符合其人设的分析论述。

所有投票完全独立，巴菲特看不到索罗斯的观点，格雷厄姆看不到伍德的判断。分歧之所以真实，正因为每一份思考都是独立完成的，没有互相影响。

最多支持10位大师并行推理，用速度快的模型时，整个议事过程不到一分钟就能完成。

### 第三幕 · 神谕（The Augury）

高级合成模型会通读所有投票结果，权衡不同投资流派的观点权重，生成最终的**神谕**：一句话总结结论，加上一份平衡的分析叙事，覆盖共识点、核心分歧、最有冲击力的少数派意见，以及可能会让大师们改变判断的关键催化剂。

完整的议事记录——包括每位大师的独立推理过程、按流派统计的投票分布、被提及最多的理由和担忧，都会保存为Markdown报告，你可以直接阅读、分享，或者整合到自己的分析流程里。

## 议事会名册

十八位大师分属五大投资流派：

| 流派 | 大师名单 |
|---|---|
| **价值派** | 巴菲特 · 芒格 · 格雷厄姆 · 克拉曼 · 马克斯 · 格兰瑟姆 |
| **成长派** | 林奇 · 费雪 · 凯茜·伍德 · 尼克·斯利普 |
| **宏观派** | 索罗斯 · 达利欧 · 德鲁肯米勒 |
| **量化派** | 西蒙斯 · 阿斯尼斯 |
| **逆向派** | 邓普顿 · 德雷曼 · 迈克尔·伯里 |

每位大师的人设都保存在一个简单的YAML文件里，描述其核心投资哲学、看重的指标、避之不及的雷区，以及说话风格。新增一个自定义大师只需要十行配置。

## 快速开始

```bash
uv tool install git+https://github.com/xgzlucario/augur.git
augur list-personas        # 确认十八位大师全部就位
augur run AAPL             # 召集完整议事会
```

Augur需要两组凭证：大模型服务商密钥和搜索服务商密钥。在你运行Augur的目录下新建一个`.env`文件，填入以下内容：

```env
OPENAI_API_KEY=sk-...
OPENAI_BASE_URL=                   # 官方OpenAI留空，其他兼容服务商填完整URL
OPENAI_MODEL_RESEARCH=gpt-4o-mini  # 便宜速度快，每位大师调用一次
OPENAI_MODEL_SYNTHESIS=gpt-4o      # 能力更强，每次完整运行调用3次

EXA_API_KEY=                       # 从 https://exa.ai 获取
TAVILY_API_KEY=                    # 从 https://tavily.com 获取
```

所有兼容OpenAI API格式的服务商都可以使用。一次完整运行通常需要1-2分钟，Token成本约1美元，加上少量搜索调用费用，报告默认保存在`./reports/<标的代码>_<YYYY-MM-DD>.md`。

## 用法说明

```bash
augur run AAPL                              # 完整议事会投票
augur run TSLA --limit 5                    # 仅使用前5位大师
augur run NVDA --schools value,contrarian   # 限定使用指定投资流派
augur run BTC --concurrency 5               # 降低并发数，适配有速率限制的服务商
augur run AAPL --lang zh                    # 输出中文叙事
augur run AAPL -v                           # 开启详细日志
```

`--lang`参数支持`en`（默认）、`zh`、`ja`、`ko`、`es`、`fr`、`de`、`pt`、`ru`等，以及你的大模型能识别的任意语言。自由文本字段会按指定语言输出，结构性字段（操作建议、仓位、标的代码）仍保持英文，方便下游工具兼容。

## 添加自定义大师

在`personas/<流派名>/<id>.yaml`下新建YAML文件：

```yaml
id: my_master
name: 我的自定义大师
school: value              # 可选值: value | growth | macro | quant | contrarian
philosophy: |
  一到两句话描述其核心投资理念。
key_metrics:
  - 其最看重的指标列表
avoids:
  - 其绝对不会碰的雷区列表
voice: |
  描述其说话语气、口头禅、为人熟知的金句风格。
```

下次运行`augur list-personas`时，新的大师会自动被识别。所有人设的`id`必须全局唯一。
